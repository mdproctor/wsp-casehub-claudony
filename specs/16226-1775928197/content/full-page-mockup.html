The problem

Claude Code dies
when you disconnect

Close the terminal. Switch devices. Laptop goes to sleep. Your session is gone — the context, the working state, everything. You're back to square one.

And there's no way to reach it from your iPad, your phone, or another machine. Claude Code is tethered to wherever it was started.

How it works

A colony that lives
independently of you

Two lightweight processes. Sessions in tmux. You connect from any browser.

Browser / PWA
Any device with a browser. Install as a PWA for a full-screen native feel on iPad.
Controller Claude
A Claude instance with MCP tools. Orchestrates the entire colony — create, read, kill sessions with natural language.
iTerm2 / Terminal
Local terminal integration. Open any session directly as an iTerm2 window.
Session — api-dev
A live tmux pane. Claude Code running your API project. Available to browser or iTerm2.
Session — frontend
Another tmux pane. Completely independent. Persists whether connected or not.
Session — dormant
Waiting. Process lives in tmux. Reconnect instantly whenever you need it.

Get started

Colony running in three steps

Download the binary
Grab the latest release for macOS. A single native binary — no JVM required. Fast startup, low memory.
Start the server
Run the server on your Mac. It manages tmux sessions and serves the web dashboard. Open the URL in any browser.
Connect the agent
Start the agent locally to give your controller Claude MCP access to the full colony.
# Start the server (Mac Mini / always-on machine)
$ ./claudony server --bind 0.0.0.0
 
# Start the agent (your MacBook)
$ ./claudony agent --server http://mac-mini.local:7777
